  3. HTML 用 标签对 添加语义

二 HTML骨架

  1. Doctype HTML4.01 XHTML1.0

    1. strict 有一些标签不能用
    2. transitional loose 普通
    3. frameset frameset 代替body 其他等于transitional

  2. head

    1. meta

      1. 字符集 charset

        1. utf8

        2. GB2312

      2. Keywords

      3. Description

    2. title

    3. style

  3. body

三 HTML语法特性

  1. 对换行 tab缩进不敏感

  2. 空白折叠现象 文字间多个空格 tab 换行 都被折叠为一个空格

  3. 标签要严格封闭

四 h 标签 p 标签

  1. h不是嵌套

  2. p只能包裹文字 图片 表单元素

    1. Chrome 审查元素 F12

  3. 有两种 容器级和文本级

    1. 容器级可以放任何的东西
    2. 文本级只能放文字 图片 表单元素

五.图片 img

  1. 能插入的类型 jpg gif png bmp
  2. img src=”美景1.jpg” alt=”” 用<>括起来
  3. 单标签 自封闭标签 不需要加语义
  4. src是属性 alt 图片找不到时显示的替代图片
  5. 相对路径

六.超级链接 a href

  1. 代码anchor href title target

  2. title 鼠标悬停文本

  3. target= “_blank” 在新标签页打开

  4. 链接文字 在两个标签之间的文字

  5. 发光旋转css

6.页面内锚点

7.跨页面锚点

8.图片链接 图片尺寸 绝对路径网络图片

七.无序列表 ul li

  1. 组标签

  2. 里面的直接子标签必须是li

  3. li必须被ul包裹

  4. li里面的内容可能非常多

  5. li是个容器级标签 什么都能放

  6. 多级列表

八.有序列表 ol li

  • 用的少,用无序列表加数字更好,代码块后面再加不好加

九.定义列表 dl dt dd

  • dt、dd都是容器级标签，想放什么都可以

十.div

  • 1.division 分割 不增加分割但是增加语义 容器级
  • 2.div+css div复责内容 结构 分块
  • 3.css复责样式 布局
  • 4.div容器级标签

十一.span

span 范围 文本级标签

十二.表单 form input

1.form

  • 1.表单就是收集用户信息的，就是让用户填写的、选择的。
  • 2.action属性就是表示，表单将提交到哪里。 method属性表示用什么HTTP方法提交，有get、post两种。

2.文本框 text

3.密码框 password

4.单选按钮 radio

5.复选框 checkbox

  • 复选框，最好也是有相同的name

6.下拉列表 select option

7.多行文本框(文本域) textarea

8.普通按钮 button

9.提交按钮 submit

  • 表单会被提交到，form标签的action属性中的那个页面中去。

10.重置按钮 reset

11.标签 label for id
